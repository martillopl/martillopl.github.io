Tema 7. Administración de sistemas — Cuadro comparativo

Tabla pensada para distinguir conceptos que suelen confundirse en examen. Se resume qué hace cada tecnología, en qué se basa y para qué se usa.

1. UNIX/Linux vs virtualización

Aspecto UNIX/Linux Virtualización
Naturaleza Sistema operativo de servidor. Tecnología para ejecutar varios sistemas sobre un mismo hardware.
Función Gestionar recursos, procesos, usuarios y servicios. Aislar y consolidar cargas.
Elemento clave Consola, servicios y automatización. Hipervisor.
Idea clave Base del servidor. Reparto del hardware entre sistemas lógicos.

2. Virtualización vs contenedores

Aspecto Virtualización Contenedores
Unidad aislada Máquina virtual completa. Aplicación con sus dependencias.
Recursos Más pesados; cada VM incluye sistema operativo. Más ligeros; comparten el núcleo del anfitrión.
Arranque Más lento. Más rápido.
Idea clave Aísla sistemas. Aísla aplicaciones.

3. SAN vs NAS

Aspecto SAN NAS
Nivel de acceso Bloque. Fichero.
Uso típico Bases de datos, virtualización y entornos críticos. Compartición de archivos y copias de seguridad.
Rendimiento Alto y baja latencia. Más dependiente del tráfico de red.
Idea clave Almacenamiento cercano al servidor. Almacenamiento compartido por red.

4. Backup vs replicación

Aspecto Backup Replicación
Función Conservar una copia para restaurar. Mantener una copia sincronizada en otro sistema.
Objetivo Recuperar datos tras pérdida o error. Reducir impacto de una caída y mejorar continuidad.
Relación temporal Punto en el tiempo. Casi en tiempo real o con retraso controlado.
Idea clave Guarda una versión. Duplica el estado.

5. Cluster vs servidor aislado

Aspecto Servidor aislado Cluster
Composición Un único servidor presta el servicio. Varios servidores cooperan como un sistema.
Disponibilidad Más vulnerable al fallo único. Más tolerante a fallos.
Uso Entornos simples o no críticos. Alta disponibilidad, balanceo o procesamiento distribuido.
Idea clave Un nodo. Varios nodos coordinados.

6. Alta disponibilidad vs recuperación ante desastres vs continuidad de negocio

Aspecto Alta disponibilidad Recuperación ante desastres Continuidad de negocio
Objetivo Evitar o minimizar caídas. Restaurar sistemas tras un incidente grave. Garantizar que la actividad siga o se recupere a tiempo.
Ámbito Técnico e inmediato. Técnico y de emergencia. Técnico, organizativo y operativo.
Horizonte Corto plazo. Tras el desastre. Más amplio y estratégico.
Idea clave Que no se caiga. Que vuelva a arrancar. Que el negocio no se pare.

Resumen mental rápido: virtualización = máquinas completas; contenedores = aplicaciones empaquetadas; SAN = bloques; NAS = ficheros; backup = copia recuperable; replicación = copia sincronizada; cluster = varios servidores; HA = continuidad del servicio; DR = recuperación tras desastre; continuidad = supervivencia del negocio.
