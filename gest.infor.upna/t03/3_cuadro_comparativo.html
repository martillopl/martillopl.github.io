Tema 3. Fundamentos de programación — Cuadro comparativo

Tabla pensada para fijar diferencias que suelen confundirse en examen. La idea es repasar en bloque y localizar qué hace cada tecnología.

1. Tipos de datos vs variables

Aspecto Tipo de dato Variable
Qué es Clasificación del valor. Nombre o contenedor que almacena un valor.
Función Define qué operaciones son válidas. Permite guardar y usar datos en el programa.
Ejemplo Entero, cadena, booleano. edad, nombre, activo.
Idea clave Qué clase de dato es. Dónde se guarda ese dato.

2. Programación estructurada vs orientación a objetos

Aspecto Programación estructurada Programación orientada a objetos
Unidad básica Bloque de instrucciones. Objeto.
Forma de organizar Secuencia, selección e iteración. Clases, objetos y relaciones.
Ventaja principal Claridad y simplicidad lógica. Reutilización, modularidad y mantenimiento.
Idea clave Resuelve por pasos. Resuelve modelando entidades.

3. Patrón de diseño vs framework

Aspecto Patrón de diseño Framework
Naturaleza Solución conceptual reutilizable. Conjunto de herramientas y estructura de desarrollo.
Dependencia No es una librería ni un producto. Sí es una tecnología concreta.
Objetivo Resolver un problema recurrente de diseño. Acelerar y ordenar el desarrollo.
Idea clave Cómo diseñar. Con qué construir.

4. Java vs JavaScript

Aspecto Java JavaScript
Tipo de lenguaje Orientado a objetos, de propósito general. Lenguaje de scripting muy ligado a la web.
Ejecución Bytecode sobre la máquina virtual Java. Principalmente en navegador, también en servidor.
Uso típico Aplicaciones, servidores, entornos corporativos. Interactividad web y aplicaciones frontend.
Idea clave Portabilidad y robustez. Interacción dinámica en web.

5. HTML vs CSS vs JavaScript

Aspecto HTML CSS JavaScript
Función Estructura y contenido. Presentación visual. Comportamiento e interactividad.
Qué controla Elementos del documento. Color, tamaño, diseño, disposición. Eventos, lógica y respuesta a acciones.
Rol en la web Base semántica. Capa de estilo. Capa dinámica.
Idea clave Esqueleto. Apariencia. Interacción.

6. IDE vs editor de texto

Aspecto Editor de texto IDE
Función Editar código. Editar, compilar, ejecutar y depurar.
Alcance Más simple. Más completo e integrado.
Ayudas Limitadas. Autocompletado, navegación y depuración.
Idea clave Escribe código. Gestiona el ciclo de desarrollo.

7. Control de versiones vs gestor de dependencias

Aspecto Control de versiones Gestor de dependencias
Qué gestiona Cambios del código en el tiempo. Librerías y paquetes del proyecto.
Problema que resuelve Colaboración, trazabilidad y recuperación. Instalación y coherencia de bibliotecas.
Ejemplo habitual Git. Maven, npm o equivalentes.
Idea clave Controla el código. Controla las dependencias.

8. Metodología tradicional vs ágil

Aspecto Tradicional Ágil
Planificación Más cerrada al inicio. Iterativa y adaptable.
Gestión del cambio Más costosa. Más natural y continua.
Trabajo en equipo Por fases más separadas. Colaboración continua.
Idea clave Plan fijo. Adaptación continua.

Resumen mental rápido: dato = tipo, código = estructura o lógica, objeto = entidad con estado y comportamiento, HTML = contenido, CSS = estilo, JavaScript = interacción, IDE = entorno completo, Git = versiones, gestor = librerías.
