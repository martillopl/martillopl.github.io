Tema 4. Arquitectura de aplicaciones — Cuadro comparativo

Comparativa centrada en diferencias que suelen confundirse en examen. La idea es distinguir funciones, objetivos y alcance de cada concepto.

1. Monolítica vs microservicios

Aspecto Arquitectura monolítica Arquitectura de microservicios
Estructura Una sola aplicación integrada. Conjunto de servicios pequeños e independientes.
Despliegue Unidad única. Cada servicio puede desplegarse por separado.
Escalabilidad Más difícil de escalar de forma selectiva. Más flexible para escalar por partes.
Idea clave Todo junto. Separado y distribuido.

2. Servidor web vs servidor de aplicaciones

Aspecto Servidor web Servidor de aplicaciones
Función principal Atender peticiones web y servir contenido. Ejecutar lógica de negocio y servicios comunes.
Alcance Más cercano a la capa de presentación. Más cercano a la lógica de negocio.
Servicios habituales HTTP, contenido estático, publicación web. Sesiones, transacciones, seguridad, integración.
Idea clave Entrega páginas y respuestas. Ejecuta componentes de la aplicación.

3. Alta disponibilidad vs rendimiento

Aspecto Alta disponibilidad Rendimiento
Objetivo Que el servicio siga disponible ante fallos. Que el servicio responda rápido y con eficiencia.
Técnicas habituales Redundancia, réplica, clustering, balanceo. Optimización, caché, ajuste de recursos.
Enfoque Resiliencia. Velocidad y capacidad de respuesta.
Idea clave Evitar caídas. Evitar lentitud.

4. SOAP vs REST

Aspecto SOAP REST
Naturaleza Protocolo más rígido. Estilo arquitectónico más ligero.
Formato Muy asociado a XML. Puede usar varios formatos, frecuentemente JSON.
Uso típico Integración empresarial formal. APIs web modernas.
Idea clave Más estructurado. Más flexible y simple.

5. Aplicación web vs aplicación móvil

Aspecto Aplicación web Aplicación móvil
Dispositivo Se usa principalmente en navegador. Diseñada para teléfono o tableta.
Condicionantes Pantalla y navegador. Pantalla pequeña, batería, tacto y conectividad variable.
Arquitectura Más orientada a web y portal. Más adaptada al contexto de movilidad.
Idea clave Acceso vía web. Uso nativo o adaptado a movilidad.

6. Responsive design vs sitio móvil separado

Aspecto Responsive design Sitio móvil separado
Modelo Una sola web que se adapta. Versiones distintas para dispositivos distintos.
Mantenimiento Más unificado. Más duplicado y costoso.
SEO Suele ser más sencillo de gestionar. Puede requerir más trabajo de optimización.
Idea clave Una interfaz adaptable. Una interfaz específica por dispositivo.

7. Portal corporativo vs web informativa

Aspecto Portal corporativo Web informativa
Función Punto central de acceso a servicios y aplicaciones. Publicar y difundir información.
Interacción Alta, con personalización y acceso integrado. Más orientada al contenido que al servicio.
Usuarios Empleados, estudiantes, proveedores o ciudadanía. Visitantes o lectores de contenido.
Idea clave Acceso a servicios. Difusión de información.

8. Gestión de contenidos vs despliegue de contenido

Aspecto Gestión de contenidos Despliegue de contenido
Qué hace Organiza la creación, publicación y mantenimiento editorial. Lleva el contenido al entorno de publicación.
Enfoque Editorial y organizativo. Técnico y operativo.
Control Permisos, flujos, versiones, estructura. Actualización y disponibilidad en el sitio.
Idea clave Gestionar qué se publica. Publicarlo correctamente.

Resumen mental rápido: monolito = una pieza; microservicios = piezas independientes; SOAP = más rígido; REST = más ligero; responsive = una web adaptable; portal = acceso central; alta disponibilidad = continuidad; rendimiento = rapidez.
